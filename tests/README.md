# tests

**Purpose:** Unit and integration tests so students can verify each component and refactor safely.

**What students will implement:** Use pytest (or unittest). Test individual tools (with mocks), schemas (validation), and optionally the validation service or agent with a mock LLM. Keep tests fast and focused.

**Weeks:** 2 (schema tests), 3 (tool tests), 6 (integration). Add `pytest` to requirements.txt when you start testing.

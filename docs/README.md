# docs

**Purpose:** Architecture diagrams, design decisions, and runbooks — so the project is understandable and maintainable.

**What students will implement:** Add a short architecture doc (MCP flow diagram), API contract (endpoints + schemas), and any “why we chose X” notes. Use Markdown and optional Mermaid diagrams.

**Weeks:** 1 (skeleton), 5–6 (detail as you integrate).
